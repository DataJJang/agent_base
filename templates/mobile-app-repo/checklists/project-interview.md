# Project Interview Checklist

## 1. 기본 식별

- [ ] 프로젝트명
- [ ] 저장소명
- [ ] 프로젝트 목적
- [ ] 프로젝트 패밀리
- [ ] 프로젝트 성격
- [ ] 저장소 구성 방식
- [ ] 대상 사용자

## 2. 실행 대상

- [ ] 대상 플랫폼
- [ ] 런타임 역할
- [ ] 동작 OS
- [ ] 단일 저장소 또는 멀티 저장소 여부

## 3. 기술 선택

- [ ] 언어
- [ ] 프레임워크
- [ ] 런타임 버전
- [ ] 빌드 도구
- [ ] 테스트 도구
- [ ] 데이터 저장소
- [ ] cache
- [ ] 서비스 기동 형태
- [ ] 로깅 방식

## 4. 운영 선택

- [ ] 배포 유형
- [ ] 보안 또는 인증 방식
- [ ] 핵심 외부 연동
- [ ] 대상 환경

## 5. 확정 검증

- [ ] 인터뷰 결과가 `project-generation-spec`으로 변환 가능하다
- [ ] `stack-matrix` 권장 기본값과 충돌하지 않는다
- [ ] 예외가 있으면 overlay 필요 여부를 기록했다
- [ ] 템플릿 선택이 `project-selection-mapping`과 정합하다
