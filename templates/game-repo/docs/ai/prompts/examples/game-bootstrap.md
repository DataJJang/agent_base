# Game Bootstrap Example

```text
Run a project bootstrap interview for a new game repository.

Repository: my-unity-game
Branch: dev
Goal: bootstrap a Unity-based game prototype repository
Target environments: local, dev
Preferred project family: game
Expected runtime roles: client, tooling
Out of scope:
- backend-service implementation

Ask the user for:
- project name and purpose
- target platforms such as Windows, macOS, Android, iOS
- Unity versus other engine choice
- runtime role split
- local-only prototype versus production-ready direction
- logging and build validation approach

Then produce a normalized project generation spec.
```
