# Parity Validation

## 1. 목적

전환 후에도 기존 시스템과 동일한 기능, 계약, 데이터 결과를 유지하는지 검증하는 기준이다.

## 2. parity 유형

- 기능 parity
- API contract parity
- 데이터 parity
- 배치 side-effect parity
- receiver routing parity
- 운영/배포 parity

## 3. 최소 산출물

- parity 대상 목록
- before/after validation 방법
- sample input/output
- 허용 차이와 비허용 차이
- cutover 전 최종 검증 기준
